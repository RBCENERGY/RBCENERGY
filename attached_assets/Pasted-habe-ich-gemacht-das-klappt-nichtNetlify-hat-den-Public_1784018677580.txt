Wouter macht das Routing im Browser, aber Netlify sucht bei direktem Aufruf von /leistungen nach einer echten Datei und findet keine.
Fix: Eine Datei _redirects muss in den Build-Output. Am saubersten legst du sie in Replit im public-Ordner des Projekts an, dann kopiert Vite sie automatisch mit in den dist-Output.
Prompt für Replit:

Erstelle im Projekt artifacts/rbc-website eine Datei namens _redirects (ohne Dateiendung) im public-Ordner. Falls es keinen public-Ordner gibt, lege ihn im Projekt-Root an (auf derselben Ebene wie index.html bzw. src), sodass Vite den Inhalt beim Build automatisch nach dist kopiert.
Inhalt der Datei _redirects (genau eine Zeile):
/*    /index.html   200
Diese Regel sorgt dafür, dass alle Pfade an index.html ausgeliefert werden, damit das clientseitige Routing (Wouter) auch bei direktem Aufruf von Unterseiten wie /leistungen funktioniert, statt einen 404 zu zeigen.
Prüfe, dass die Datei nach npm run build tatsächlich im Build-Output landet (im dist- bzw. dist/public-Ordner als _redirects). Baue das Projekt anschließend neu und bestätige, dass die Datei im Output enthalten ist.
